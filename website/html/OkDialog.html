DuxOkDialog

The <DuxOkDialog> component serves as a convenient dialog when all you need is an Okay (and optionally a Cancel) button. In addition to the properties recognized by DuxPanel and DuxDialog, the following are properties recognized by DuxOkDialog.

Property Default Value Description
allowClose true True if a close button is rendered on the right side of the title bar.
allowEnter true If true, if the user presses Enter it is handled as if they clicked the Okay button.
allowEsc true If true, if the user presses Escape it is handled as if they clicked the Cancel button.
cancelClassName (none) The CSS class(es) used for the Cancel button.
okClassName (none) The CSS class(es) used for the Okay button.
okDisabled false If true, the Okay button is disabled.
onCancel (none) Called when the user clicks the Cancel button.
onOk (none) Called when the user clicks the Okay button.
shouldClose (none) If present, this function is called when the user clicks the Okay button. It returns true if the user is allowed to close the dialog with the Okay button. This can be used to verify the user has completed a necessary interaction in the dialog. For example, a checkbox to accept terms.
showCancel false If true, a Cancel button is rendered to the right of the Okay button.
