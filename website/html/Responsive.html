Responsive Panels

<DuxPanel> is designed to be responsive from Desktop browsers down to mobile devices. This is accomplished using a set of sizing breakpoints similar to Twitter's Bootstrap CSS framework.

Sizing Breakpoints

There are five sizing breakpoints, defined to be exactly the same as Bootstrap. This simplifies web page design if your site is already using Bootstrap. These breakpoints are:

Breakpoint Description
xs Extra small devices (portrait phones, less than 576px)
sm Small devices (landscape phones, 576px and up)
md Medium devices (tablets, 768px and up)
lg Large devices (desktops, 992px and up)
xl Extra large devices (large desktops, 1200px and up)

Dimensions and Placement

Panels are sized and located on the web page using one of three possible values: a number (actual pixels), a percent, or an object detailing sizes at specific breakpoints. If an object is supplied, the size at each breakpoint can be a number (actual pixels) or a percent. These values are recognized in the <DuxPanel> properties of left, top, width, and height. Here are some examples of this: 

    <DuxPanel width="50%" height={150} top={50}>...</DuxPanel>

Sizes and placement in breakpoints look like this:

    const width = {
        xs: "100%",
        sm: "90%",
        md: 400   // Numbers or percents can be used for each breakpoint
    };

    <DuxPanel width={width} ...

Notice the width object is missing lg and xl. If a size breakpoint is missing from the object, the breakpoint for the next smallest screen is used. In this case, browsers with large or extra-large windows would fall back to the medium breakpoint. If the browser window is the xs size and that breakpoint is missing from the object, the next lowest breakpoint is used instead.
